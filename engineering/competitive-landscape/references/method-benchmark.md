# Method Benchmark

This skill should stay aligned with high-confidence public methods for player
discovery, technical competition mapping, and scouting workflows.

## External Benchmark Sources

- WIPO, *Guidelines for Preparing Patent Landscape Reports* (2015)
- WIPO, *The Patent Analytics Handbook* (2022)
- OECD, *OECD Patent Statistics Manual* (2009)
- DHS S&T, *Technology Scouting* process description
- NASA technology scouting benchmarking report (2023)

## Top-Tier Method Requirements

1. Build a candidate pool before prioritizing players.
2. Separate:
   - discovered players
   - shortlisted players
   - deep-dived players
3. Use multiple evidence families for tiering:
   - patents
   - papers
   - product/deployment/public signals
4. Keep sector-level and player-level evidence distinct.
5. Keep white-space claims strict:
   - sparse
   - valuable
   - entry path visible
6. Keep the process reproducible through:
   - query log
   - source index
   - claim ledger

## What "Leading" Looks Like In This Skill

- the skill discovers before it ranks
- the skill does not confuse fame with technical position
- sector conclusions do not overwrite per-player evidence
- downgraded-tool runs do not pretend to be complete market maps

## Upgrade Triggers

Refresh the skill when one of these happens:

- player discovery repeatedly misses non-obvious actors
- white-space output becomes too generic or inflated
- a new public source improves domain-specific player evidence materially
