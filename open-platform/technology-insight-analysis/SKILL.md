# 技术竞争洞察报告

## 目的

使用本 Skill 将已完成标签化的技术专利数据转化为管理层可读的竞争技术分析报告。分析必须以证据为基础，表达方式适合商业决策，并严格遵守数据边界。

生成完整报告或提示词模板时，读取 [references/white-paper-framework.md](references/white-paper-framework.md)，使用其中的白皮书框架和可复用提示词。

生成 HTML 可视化报告时，读取 [references/html-report-template-spec.md](references/html-report-template-spec.md)，遵循其中关于版式、色彩系统、图表清单、CSS 变量和 ECharts 配置的要求。

## 端到端业务流程

本 Skill 是适用于任意技术领域的多步骤流程中的最后分析环节：

1. **专利检索**：使用申请人优先或主题优先的检索 Skill，收集目标专利范围。
2. **筛选**：过滤专利，保留与目标技术边界相关的记录。
3. **标引/打标签**：对保留专利应用领域标签，如技术类型、功能、工艺、材料等。
4. **洞察分析（本 Skill）**：仅在专利已检索、筛选和标引后使用本 Skill；将标引结果作为主要分析基础。

如果用户只提供公司名称，先建议或执行专利检索。如果用户提供的是未筛选结果，先建议筛选。如果提供的是已保留但未标引的专利，先建议标引。除非数据中已有正式标签，否则不得把报告写得像上游步骤已经完成。

## 输出格式判断

当用户要求分析报告时，按以下规则选择输出格式：

- **Markdown 草稿**：用户要求草稿、大纲或仅需内容时默认使用。
- **HTML 可视化报告**（用户提到 `HTML格式`、`可视化报告`、`单页滚动报告`）：生成完全自包含的单页 HTML 文件，包含 ECharts 交互图表、专业视觉风格和滚动式导航，并遵循 [references/html-report-template-spec.md](references/html-report-template-spec.md) 的完整规范。

**正式交付默认格式**：当用户要求“完整报告”或“最终版”但未指定格式时，默认生成 HTML 版。

## 核心规则

写任何分析前，先应用以下规则：

1. 只分析已由提供数据中的标签字段支撑的维度。
2. 不得推断、发明或强行构造未标引的维度。
3. 如果某个维度数据不足以闭合逻辑链，应从正文深度章节中省略，只在研究边界或后续数据建议中提及。
4. 区分 **专利数据结论** 和 **产业映射参考**。
5. 行业知识只能作为技术类型/功能映射参考，不能作为竞争者活动的直接证据。
6. 每条战略结论至少引用一种依据：标签字段统计、专利文本、时间趋势、法律状态、同族/被引指标，或明确标注的产业映射。
7. 除非证据非常强，避免绝对化表述。优先使用“可能表明”“提示”“与……一致”“仍需进一步验证”等措辞。
8. 关键结论必须标注证据强度：强、中、弱。
9. 写作风格应接近高端商业白皮书，而不是原始数据库统计报告。

## 最低字段门禁

起草前，检查可用字段并判断哪些章节有数据支撑：

- **技术类型/类别**：支撑技术路线布局、资源集中度和类型热力图。
- **技术功能**：支撑功能价值布局、功能热点和技术-功能矩阵。
- **工艺/方法**：支撑技术路径、工艺路线集中度和工艺能力讨论。
- **申请人/公司**：支撑竞争者比较和公司战略画像。
- **申请/公开年份**：支撑趋势分析和未来关注信号。
- **专利状态/法律状态**：支撑有限的法律质量与风险讨论。
- **同族/被引数据**：支撑更强的壁垒和专利价值讨论。

如果字段不存在，不得创建相应深度分析章节。

## 推荐分析框架

优先使用“技术类型 × 技术功能”作为核心分析框架。

当数据支持时，应包含：

- 总体竞争格局
- 技术类型资源定位
- 技术功能价值定位
- 技术类型 × 功能矩阵
- 存在工艺标签时的工艺/方法路线分析
- 竞争者战略画像
- 潜在壁垒与机会窗口
- 未来关注方向（仅当时间字段存在）
- 可执行的研发与专利布局建议

## 标准章节结构（HTML 版）

根据实际技术领域和可用标签维度调整章节结构：

| 章节 | section id | 内容 |
|---|---|---|
| 执行摘要 | summary | 核心战略结论 |
| 第一章 | overview | 竞争格局总览 |
| 第二章 | tech-type | 技术类型布局分析 |
| 第三章 | function | 功能价值布局分析 |
| 第四章 | matrix | 核心矩阵分析 |
| 第五章 | profiles | 竞争对手战略画像 |
| 第六章 | opportunity | 壁垒识别与机会窗口 |
| 第七章 | action | 研发与专利布局建议 |

## 白皮书写作风格

除非用户另有要求，使用中文写作。语言应正式、克制，并符合战略咨询报告表达。

每个正文章节都应包含：

- 数据发现
- 战略解读
- 对目标公司的意义
- 证据强度

## 输出边界

不得：

- 把专利数量等同于技术壁垒。
- 在缺少法律质量字段时，把高密度布局等同于高法律风险。
- 在缺少产品/场景字段时，宣称产品或市场定位。
- 把工艺标签当作完整制造链证据。

## 日期政策

报告页眉、版本日期和页脚始终使用当前年份。报告必须显示 **2026年**。
