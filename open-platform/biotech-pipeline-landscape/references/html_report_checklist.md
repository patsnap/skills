# HTML报告生成检查清单

## 一、生成前核查（数据完整性）

### P0 级别（必须完成，否则不可交付）
- [ ] 所有临床登记号已核实，或标注"待补充（CDE/ChiCTR/NCT）"
- [ ] 已上市CAR-T产品获批日期有监管来源，2024年后标注"请核实"
- [ ] 无"FTO风险降低X%/Y%"等无依据量化表述
- [ ] 无"CDR差异≥1处即可形成独立IP空间"等过强结论
- [ ] 所有关键数据附 [S#] 来源标记

### P1 级别（重要，影响报告完整性）
- [ ] 包含 CAR-T 专属8模块评分表（A–H模块，1–5分制，4维度输出）
- [ ] 包含目标品种构型信息表（已知字段 + 待补充字段）
- [ ] 包含序列查询8步SOP
- [ ] 包含序列命中→FTO映射表
- [ ] 包含核心专利同族地图（≥7项专利）

### P2 级别（优化项）
- [ ] 制造可及性分析
- [ ] CAR-T/基因治疗通用监管CMC框架
- [ ] 真实世界证据与自免适应症拓展
- [ ] 智慧芽5维持续监控主题 + 预警事件表
