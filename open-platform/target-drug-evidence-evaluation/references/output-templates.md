# Output Templates — RD&BD Evaluate Target×Drug×Evidence×Patent

本文件定义本 skill 的中文输出模板。所有输出必须严格按照以下四块结构呈现。

> **禁止**：不得在四块结构之外额外输出情报摘要、新闻汇编或事件时间线。
> **必须**：所有有评分的行必须有对应的「证据」列，不得留空。

---

## 输出块 1 — Executive Verdict（执行判断）

```
## 🎯 立项分诊结论 — [靶点名] × [适应症/广谱]

**最终判断：[Go / Watch / Niche Go / No-Go / Insufficient Evidence]**

- **结论**：[1-2 句话说明判断结果]
- **一句话理由**：[最核心的支持或否定理由，不超过 30 字]
- **关键假设**：[本次评估依赖的 1-3 个核心假设]
- **最大风险**：[当前最不确定或最可能推翻结论的单一因素]
```

---

## 输出块 2 — Scorecard（评分卡）

```markdown
## 📊 Scorecard

| 维度 | 分数 | 判断 | 证据 |
|---|---:|---|---|
| Biology Confidence（生物学置信度） | /20 | 强/中/弱/不足 | [证据来源简述] |
| Translational Readiness（转化就绪度） | /15 | 强/中/弱/不足 | [证据来源简述] |
| Clinical Tractability（临床可行性） | /15 | 强/中/弱/不足 | [证据来源简述] |
| Competitive Whitespace（竞争空白度） | /20 | 强/中/弱/不足 | [证据来源简述] |
| IP / FTO Risk（知识产权风险） | /15 | 低/中/高/极高 风险 | [证据来源简述] |
| Strategic Fit（战略契合度） | /15 | 强/中/弱/不足 | [与 user_type/stage_intent 匹配说明] |
| **合计** | **/100** | | |
```

**填写规则**：
- 每行「分数」列填写整数
- IP/FTO 维度若专利工具不可用，分数填 `N/A`，判断列填 `数据缺失`
- 「证据」列必须引用具体工具返回数据

## 输出块 3 — Evidence Matrix（证据矩阵）

```markdown
## 🔬 Evidence Matrix

| 证据类型 | 发现 | 对立项影响 | 数据来源 |
|---|---|---|---|
| 靶点生物学 | [关键发现] | 正面影响 ✅ / 负面影响 ⚠️ / 中性/待定 ➖ | ls_target_fetch |
| 临床前转化 | [关键发现] | ... | ls_translational_medicine_search |
| 临床活跃度 | [在研试验数，阶段分布] | ... | ls_clinical_trial_search |
| 临床结果 | [成功/失败/混合] | ... | ls_clinical_trial_result_search |
| 竞争格局 | [同靶点药物数，领先者] | ... | ls_drug_search |
| 里程碑失败信号 | [discontinued/suspended 事件] | ... | ls_drug_milestone_fetch |
| 治疗指南 | [靶点/适应症在指南中的地位] | ... | ls_clinical_guideline_vector_search（若可用）|
| 专利布局 | [专利密度，活跃申请人] | ... | ls_patent_search（若可用）|
```

**填写规则**：
- 「对立项影响」固定使用：`正面影响 ✅`、`负面影响 ⚠️`、`中性/待定 ➖`
- 若某类证据工具不可用或无数据，填写 `数据缺失 — 建议补充`
- 禁止删除任何行

## 输出块 4 — Next Actions（下一步行动）

```markdown
## 🚀 Next Actions

**推荐验证动作**（按优先级排序）：
1. [最高优先级验证动作]
2. [次优先级]
3. [第三优先级]

**必须回答的尽调问题**：
- Q1：[具体尽调问题]
- Q2：[第二关键问题]
- Q3：[第三关键问题]

**建议避开的方向**：
- [明确说明哪些 modality / 适应症 / 策略路径当前应避免]
- [第二个建议避开的方向]
```

**填写规则**：
- 「推荐验证动作」至少 2 条，最多 5 条
- 「必须回答的尽调问题」至少 2 条
- 「建议避开的方向」至少 1 条，需有具体原因
- 如判断为 `No-Go`，Next Actions 聚焦于「是否存在重新立项的触发条件」
- 如判断为 `Insufficient Evidence`，Next Actions 聚焦于「哪些数据补充后可重新评估」
