# 链主企业技术联盟合作伙伴识别

## 目标

围绕一个链主企业和一个技术领域，系统识别外部技术联盟合作对象，形成面向高层决策的分析报告。报告应回答：

- 外部产业窗口是否成熟，链主现在是否应启动合作。
- 链主在目标领域有哪些能力缺口，哪些缺口应优先处理。
- 哪些企业、院所或技术公司可以补齐缺口。
- 应采用联合开发、投资并购、技术引进或自建能力中的哪类路径。
- 链主如何保持平台、数据、规则和知识产权主导权。

## 输入

用户通常提供：

- `chain_leader`：链主企业名称，例如 `徐工集团`、`三一重工`、`中联重科`。
- `domain`：技术领域，例如 `智能矿山无人驾驶`、`新能源电驱动`、`智能施工`。
- `identity`：链主身份标签，例如 `地方国企`、`央企`、`民营链主`；未提供时根据公开信息谨慎推断，并在报告中标注。
- 可选的既有资料：需求说明、行业报告、年鉴片段、候选名单、历史 HTML 报告或内部访谈纪要。

如果链主、技术领域或身份不明确，先向用户确认；不要在身份不确定时直接套用国企或民营约束。

## 核心方法论

### 产业成熟度矩阵

用 5 个维度判断外部窗口是否成立：

| 维度 | 判断问题 | 主要证据 |
|---|---|---|
| 政策 | 国家或地方政策是否明确支持 | 政府官网、部委文件、地方行动方案 |
| 标准 | 标准体系是否正在形成 | 国标公示、团体标准、行业协会材料 |
| 市场 | 真实需求或示范项目是否存在 | 蓝皮书、年鉴、竞品部署、招投标 |
| 技术 | 工程化基础是否具备 | 专利趋势、科技成果、试点项目 |
| 经济性 | 商业化条件是否成立 | 采购、运营、成本收益或客户实测数据 |

若目标领域存在强制性国家标准处于公示或制定阶段，必须在报告中标注“强制国标窗口期”，并说明链主是否应参与标准工作组。

### 能力缺口四维框架

不要只用专利数量判断链主能力缺口。按以下 4 个维度综合判断：

| 维度 | 作用 |
|---|---|
| 市场竞争差距 | 判断链主与竞品在部署规模、产品等级、客户场景上的差距。 |
| 政策/标准合规门槛 | 判断政策和标准是否要求链主补齐该能力。 |
| 运营可靠性实证 | 用年鉴、质检、事故、维护或 MTBF 等真实数据判断痛点强度。 |
| 技术布局佐证 | 用专利、论文、技术成果证明可实现性，但只作为辅助。 |

优先级规则：

- `P0`：至少 2 个维度有强证据，且包含市场竞争差距或政策/标准门槛。
- `P1`：1 个维度有强证据，或多个维度有中等证据。
- `P2`：主要只有技术趋势或专利趋势佐证，缺少市场压力和政策要求。

### 候选对象评分模型

对进入短名单的候选对象使用 5 项权重评分：

| 评分项 | 权重 | 判断重点 |
|---|---:|---|
| 缺口补齐度 | 30% | 是否能直接补齐 P0/P1 能力缺口。 |
| 规模化验证 | 20% | 是否有真实项目、客户、部署量或商业收入。 |
| 技术可集成性 | 20% | 是否能接入链主平台、供应链和产品架构。 |
| 主导权适配 | 20% | 合作后链主能否保留平台、数据、标准和规则主导权。 |
| 协同收益 | 10% | 是否扩大链主销售、服务、生态或政策影响力。 |

对国企或央企链主，主导权适配和国资合规是门槛项；主导权风险过高的候选对象不得直接进入优先推荐名单，应放入待验证对象池。

## 数据来源规则

每次分析都要覆盖 T1-T8 八类证据；缺失项要在报告中标注，不要用推测补齐。

| 编号 | 数据源 | 要求 |
|---|---|---|
| T1 | 政策 Web 检索 | 标注文号、发布机关、发布时间和核心条款。 |
| T2 | 标准 Web 检索 | 覆盖国标、团标、行业标准，标注发布/公示/制定状态。 |
| T3 | 市场 Web 检索 | 覆盖竞品部署、示范项目、招投标、商业化规模。 |
| T4 | 行业专利趋势 | 使用智慧芽或等效专利工具做全量趋势判断。 |
| T5 | 链主能力基线 | 必须使用公司树合并；失效专利要标注状态，不能当作有效能力佐证。 |
| T6 | 年鉴实证数据 | 优先使用《中国工程机械工业年鉴》等权威来源，标注页码或行号。 |
| T7 | 候选对象检索 | 结合专利和 Web，核查融资、规模、客户、整机厂合作和风险事件。 |
| T8 | 行业蓝皮书 | 引用报告名称、年份、页码或章节；优先使用最新版本。 |

数据使用要求：

- 所有关键 KPI 必须给出来源，无法核实的数字标注 `待补充` 或 `待验证`。
- 不要夸大、不捏造市场规模、部署数量、融资金额或专利数量。
- 存疑候选对象放入待验证对象池，不进入主评分表。
- 专利数量只能作为技术布局证据之一，不得单独决定缺口优先级或候选排名。

## 分析流程

1. 确认链主企业、技术领域和身份标签。
2. 制定 T1-T8 检索计划，明确每一路数据的检索式、工具、预期输出和来源标注方式。
3. 收集并整理政策、标准、市场、专利、年鉴、蓝皮书和候选对象证据。
4. 输出产业成熟度矩阵，判断紧迫度和强制国标窗口。
5. 建立链主能力基线，区分有效能力、失效布局、空白领域和待核查项。
6. 用四维框架识别 Top 3-5 能力缺口，并标注 `P0/P1/P2`。
7. 筛选候选对象，按 5 项权重评分，形成推荐名单和待验证对象池。
8. 设计合作路径，包括联合开发、股权投资、技术引进、联合标准、示范项目或自建能力。
9. 按链主身份复核主导权、国资合规、数据权属、知识产权归属和退出机制。
10. 生成 HTML 报告和结构化检索计划 JSON。

## 报告结构

默认输出 6 章：

| 章节 | 内容 |
|---|---|
| 1. 决策摘要 | 一句话结论、关键决策卡片、高层问答、行动优先级和时间轴。 |
| 2. 政策窗口与产业成熟度 | 5 维矩阵、紧迫度判断、蓝皮书市场数据、强制国标窗口。 |
| 3. 能力需求与缺口 | 市场矩阵、链主能力基线、四维证据、Top 缺口和 P0/P1/P2 分级。 |
| 4. 目标对象识别 | 候选筛选逻辑、评分模型、推荐排名、待验证对象池、主导权风险。 |
| 5. 合作路径设计 | 推荐对象、合作类型、首个交付物、协议要点和身份专项约束。 |
| 6. 行动与收益 | 决策事项、0-6 月路线图、标准工作组动作、收益、风险和待验证项。 |

## 语气规范

保持中肯、客观、面向决策，不制造危机感：

- 先肯定链主已有基础，再指出差距。
- 用“建议”“优先考虑”“值得关注”，避免“必须”“唯一正确”“严重落后”等绝对表述。
- 对候选对象优势和风险并陈，不因评分高而忽略主导权、排他合作或战略投资人风险。
- 对国企/央企链主，同时说明主导权保障和国资合规，不单边强调速度或控制权。
- 对存疑信息明确标注 `待补充`、`待验证` 或 `需人工核查`。

## 输出规范

- 输出专业 HTML 报告，包含左侧章节导航、P0/P1/P2 高亮、关键 KPI、数据来源索引和方法论展示。
- 报告末尾列出数据来源，建议按政策、年鉴、专利、蓝皮书、融资、竞品分组。
- 同时输出检索计划 JSON，包含 T1-T8 每项任务的检索目标、工具、参数、用途和预期结果。
- 对品牌色、口号、企业身份等信息，如果缺少可靠来源，应使用稳健通用样式，不要虚构官方口号。
- 样例 HTML 可以参考同目录文件 `徐工集团智能矿山技术联盟领导者挖掘报告 v7 _ 2026.05.31.html`，但新报告必须重新检索和核查，不得复用样例中的旧数据。

## 校验清单

交付前检查：

- 已覆盖 T1-T8 数据源，缺失项已标注。
- 链主专利能力基线启用了公司树合并。
- 失效专利没有被当作有效能力佐证。
- P0/P1/P2 分级有四维证据支撑。
- 候选对象评分能追溯到证据，而不是主观排序。
- 国企/央企场景已检查主导权、国资审批、数据权属和退出机制。
- 报告中没有绝对化、危机化或无来源数字。
- HTML 报告和检索计划 JSON 均已生成并可打开。
