# Report Outline

## Screen Mode

### 1. Executive Recommendation
- Proceed / proceed with conditions / hold

### 2. Novelty Boundary Summary
- Public overlap boundary (1-2 paragraphs)
- Closest prior art or prior routes

### 3. Key Risks And Blockers
- Main issues that affect the go/no-go decision

### 4. Missing Materials And Next Steps
- What data or materials are still needed
- Recommended next actions

## Review Mode

### 1. Executive Recommendation
- Bottom-line review judgment

### 2. Project Scope And Proposal Claims
- What the proposal says it is doing
- Which innovation points are under review

### 3. Novelty And Overlap
- Public overlap boundary
- Comparison table (innovation point × closest prior art × overlap degree × residual differentiator)
- Residual differentiators

### 4. Feasibility And Evidence Sufficiency
- What is supported by external evidence
- What is weak or missing

### 5. Main Risks And Material Gaps
- Key blockers
- Materials or data still needed

### 6. Recommendation And Conditions
- Proceed / proceed with conditions / hold
- Specific conditions and next steps

## Innovation Mode

### 1. Executive Summary
- Overall innovation assessment

### 2. Innovation Point Analysis
- For each point: proposal claim → external evidence → overlap → residual differentiator → judgment

### 3. Differentiation Boundary Table
- Innovation point × overlap × residual differentiator × replacement risk

### 4. Risks To Innovation Claims
- What could weaken or invalidate each claim

### 5. Recommendation
- Which points hold, which need strengthening

## Assurance Mode

### 1. Executive Recommendation
- With confidence level and evidence basis

### 2. Project Scope And Proposal Coherence
- Coherence assessment of the proposal's internal logic

### 3. Novelty And Overlap
- Detailed comparison (with separate novelty attachment)

### 4. Feasibility And Technical Decomposition
- Main technical chain, dependencies, measurement approach, failure modes

### 5. Scoring And Rating Logic
- Explicit rubric and scoring rationale

### 6. Stage-Gate Matrix
- Conditions, owners, timelines, non-fulfillment consequences

### 7. Material Gaps And Data Readiness
- What is blocking next steps

### 8. Counterevidence And Weakening Conditions
- What could change the recommendation
