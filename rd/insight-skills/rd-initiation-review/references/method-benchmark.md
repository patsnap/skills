# Method Benchmark

This skill should stay aligned with high-confidence public methods for gated
innovation review, peer review, and evidence-based project assessment.

## External Benchmark Sources

- Stage-Gate International, official gate/go-kill governance materials
- NIH, *Simplified Peer Review Framework* (2025)
- NIAID reviewer guidance and review criteria materials

## Top-Tier Method Requirements

1. Start from the project object, not from a broad topic.
2. Separate proposal-stated claims from externally supported claims.
3. Evaluate at least these dimensions:
   - importance or significance
   - rigor and feasibility
   - expertise and resources
   - evidence sufficiency
4. Make the go/kill or proceed/hold logic explicit.
5. Tie the recommendation to conditions, gaps, and next evidence needs.
6. Keep the process reproducible through:
   - query log
   - source index
   - claim ledger
   - novelty note

## What "Leading" Looks Like In This Skill

- novelty is a visible module, not an afterthought
- feasibility is judged on evidence, not optimism
- expertise/resources are treated as first-class review criteria
- missing materials are surfaced as blockers, not hidden

## Upgrade Triggers

Refresh the skill when one of these happens:

- review outputs repeatedly overfocus on novelty and underweight execution feasibility
- a new public review framework changes mainstream gated-assessment practice
- a domain repeatedly requires an additional review dimension
