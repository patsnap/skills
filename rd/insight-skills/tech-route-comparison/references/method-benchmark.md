# Method Benchmark

This skill should stay aligned with high-confidence public methods for route
comparison, readiness assessment, and technology roadmapping.

## External Benchmark Sources

- U.S. GAO, *Technology Readiness Assessment Guide* (2020)
- U.S. DOE, *Technology Readiness Assessment Guide*
- University of Cambridge IfM, T-Plan / technology roadmapping materials
- DHS S&T, *Technology Scouting* process description

## Top-Tier Method Requirements

1. Freeze the route set before broad retrieval.
2. Make the comparison basis explicit:
   - route definitions
   - dimensions
   - normalization rules
3. Evaluate maturity on evidence, not label intuition.
4. Separate:
   - route-level technical maturity
   - integration risk
   - manufacturing or deployment readiness
5. Require counterevidence and update triggers.
6. Keep the process reproducible through:
   - comparison basis
   - query log
   - source index
   - claim ledger

## What "Leading" Looks Like In This Skill

- route recommendations are scenario-specific, not abstract
- TRL is never used without explicit rubric
- hard-system routes account for integration and manufacturing readiness
- the roadmap exposes gaps, not just rankings

## Upgrade Triggers

Refresh the skill when one of these happens:

- route comparisons repeatedly collapse different comparison levels together
- maturity judgments are made without enough system-integration thinking
- a new public readiness framework materially changes best practice
