# Output Contract

## `fto_search_log.md`

Record a search summary by round. Each round must include at least:

- round number
- keyword / semantic / overlap counts
- `n_pool`
- `delta_n`
- `recall_estimate`
- `decision`
- warnings

Recommended format:

```markdown
## Round 2

| Metric | Value |
| --- | --- |
| Keyword hits | 24 |
| Semantic hits | 19 |
| Overlap | 7 |
| Pool size | 61 |
| New this round | 12 |
| Estimated recall | 0.73 |
| Decision | continue_search |
| Warnings | overlap_dependence_detected |

### Query updates
- keyword: ...
- semantic: ...
```

## `patent_pool.json`

Output must be a JSON object, not Markdown. Recommended shape:

```json
{
  "product": "example",
  "jurisdictions": ["CN", "US"],
  "legal_status_filter": "active_only",
  "records": [
    {
      "patent_no": "US123",
      "title": "Example",
      "assignee": "Example Corp",
      "jurisdiction": "US",
      "status": "active",
      "relevance_score": 4,
      "claim_breadth": "broad",
      "screening_risk": "medium"
    }
  ]
}
```

Each `records[]` entry must retain at least:

- `patent_no`
- `title`
- `assignee`
- `jurisdiction`
- `status`
- `relevance_score`
- `claim_breadth`
- `screening_risk`

## Per-patent Artifacts

For patents that enter deep analysis, generate:

- `claim_diff_matrix_[patent_no].md`
- `motivation_matrix_[patent_no].md`

Rules:

- `motivation_matrix` must reuse the missing elements from `claim_diff_matrix`
- do not build a different element-ID scheme

## `fto_report.md`

The final report must include at least these sections:

1. Matter Definition
2. Search Coverage and Method
3. Shortlisted Blocking Patents
4. Validity Attack Summary
5. Infringement and Design-Around View
6. Market-by-Market FTO Conclusion
7. Recommended Next Actions

The report must explicitly state:

- target markets
- search rounds
- cumulative unique patent count
- estimated patent universe
- estimated recall
- stopping reason

## Mandatory Disclaimer

The report must end with this disclaimer:

```text
This report was generated by the Eureka FTO Analysis Skill for technical reference only and does not constitute formal legal advice. If a high-risk patent is identified, engage qualified patent counsel for a formal FTO opinion.
```
