# Search Strategy for Novelty

Use this guide to build a candidate pool without losing the single-reference frame.

## Core Search Rule

- Search broadly first.
- Narrow only after you know which claim elements are hardest to locate.
- Keep patents and papers as the primary lane.
- Use web sources only for supplemental public-disclosure evidence or product context.

## Query Construction

Build each query from these blocks:

1. core object or process
2. unique mechanism or parameter
3. synonyms and field terms
4. optional classification, assignee, or date filters

Keep at least one high-recall query and one high-precision query active in parallel.

## Search Rounds

### Round 1: Broad recall

- use the independent-claim core terms
- avoid heavy date or assignee filters
- capture CPC/IPC classes and recurring terminology

### Round 2: Structured narrowing

- split by hard-to-find elements
- add mechanism terms, parameter ranges, or classifications
- prioritize candidates that look readable and close in technical purpose

### Round 3: Gap filling

- search specifically for still-missing elements
- use citation chains, family members, or assignee portfolios
- add web evidence only when public use, product release, or terminology needs support

## Candidate Tracking

Record for each candidate:

- source ID
- publication or disclosure date
- jurisdiction
- why it matters
- which elements it appears to cover
- fetch status

## Stop Conditions

Shift from search to reading when:

- you have one or more strong D1 candidates
- the highest-risk elements are represented in the candidate pool
- added queries mostly return duplicates or lower-quality results

If coverage is still weak, state the gap instead of pretending recall is complete.
