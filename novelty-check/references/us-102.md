# US Novelty Quick Guide

Use this guide for US novelty conclusions.

## First Check

- If the effective filing date is on or after `2013-03-16`, start with AIA `§102`.
- If earlier, flag a possible pre-AIA issue before finalizing the legal conclusion.

## Core Questions

1. What is the effective filing date?
2. Is there one reference that discloses every claim limitation?
3. Is the disclosure public before that date under `§102(a)(1)` or prior-filed under `§102(a)(2)`?
4. Is the alleged disclosure explicit, implicit, or inherent?
5. Does any grace-period argument materially affect the risk analysis?

## Practical Rules

- One readable reference can defeat novelty.
- Inherent anticipation requires necessity, not plausibility.
- Prior-filed patent applications can matter even when they were published later.
- If the claim uses a range, sequence, or control step, compare that limitation directly.

## Common Traps

- Ignoring pre-AIA vs AIA timing
- Treating a similar purpose as disclosure of the exact limitation
- Assuming inherency without a necessary technical basis
- Overlooking a possible inventor-originated grace-period fact pattern

## Output Notes

State:

- strongest D1 candidate
- which elements are disclosed
- which elements remain missing or legally debatable
- whether the result is `novelty_rejected`, `novelty_preserved`, or `uncertain`
