# Element Mapping Guide

Use this guide after a candidate reference has been fetched locally.

## Reading Order

1. `files.head` to confirm structure
2. `files.grep` to locate claim terms, synonyms, and parameter anchors
3. `files.cat` on the relevant context before writing any conclusion

Do not quote or rely on sections you have not actually read.

## Disclosure Types

- `explicit`: the feature is directly stated
- `implicit`: the feature is clearly conveyed to the skilled person from the disclosed context
- `inherent`: the feature is necessarily present even if unstated
- `missing`: no adequate disclosure located

Implicit or inherent disclosure always needs a reasoning note.

## Judgment Rules

- Ask whether the reference discloses the exact limitation, not just a nearby concept.
- Keep structural features separate from performance results unless the claim merges them.
- For ranges, state whether the reference matches exactly, overlaps, or only approaches the claimed value.
- For process steps, check order, condition, and control logic when they matter.

## Conclusion Checklist

Before marking a claim anticipated, confirm that one reference covers:

1. every limitation
2. the required relationships between limitations
3. any legally significant environment or sequence language
4. enough context to support the chosen disclosure type
