# Example Workflows

These examples show how the public skills fit together without changing each skill's core contract.

## Patentability Triage

1. Start with [../../novelty-check/SKILL.md](../../novelty-check/SKILL.md) for single-reference novelty.
2. Hand off to [../../non-obviousness-check/SKILL.md](../../non-obviousness-check/SKILL.md) when the result depends on D1 plus D2 or D3.
3. Escalate to [../../fto-analysis/SKILL.md](../../fto-analysis/SKILL.md) when the problem becomes product clearance in target markets.

## Innovation to Experiment Loop

1. Use [../../triz-analysis/SKILL.md](../../triz-analysis/SKILL.md) to frame contradictions and generate three bounded solution paths.
2. Convert the chosen path into factors and experiments with [../../doe-plan/SKILL.md](../../doe-plan/SKILL.md).
3. Keep PatSnap MCP as the default evidence source for both steps when patent or literature support matters.

## VOC to Engineering Execution

1. Run [../../qfd-analysis/SKILL.md](../../qfd-analysis/SKILL.md) on local VOC and feature inputs.
2. If the highest-priority technical conflicts need solution generation, hand off to [../../triz-analysis/SKILL.md](../../triz-analysis/SKILL.md).
3. If the result needs experimental validation, hand off to [../../doe-plan/SKILL.md](../../doe-plan/SKILL.md).

## Public to Expert Upgrade

Use the public repo when you need a transparent workflow and summary references. Move to the private companion or Eureka when you need:

- advanced case libraries
- deeper legal or technical exceptions
- richer orchestration across multiple skills
- enterprise-ready report packs
