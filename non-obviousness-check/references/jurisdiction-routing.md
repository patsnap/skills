# Jurisdiction Routing

Read this file first, then load only the jurisdiction-specific guide you need.

## Selection Order

1. Identify the target filing or enforcement jurisdiction.
2. Confirm the critical date or at least the relevant prior-art window.
3. Select one legal framework.
4. Apply that framework before writing the final inventive-step conclusion.

If any routing fact is unresolved and could change the framework, downgrade to `uncertain`.

## Jurisdiction Routing

- United States: read `references/us-103.md`

## Common Differences

- US: anchor on Graham factors, KSR rationales, and secondary considerations.
- CN: anchor on closest prior art, distinguishing features, objective technical problem, and technical teaching.
- EP: anchor on the problem-solution approach and the could-would test.
- JP: anchor on cited invention baseline, differences, motivation, and impeding factors.

## Cross-Jurisdiction Rules

- D1 stability matters before path scoring.
- A path is never strong on field similarity alone.
- Reasonable expectation of success is separate from mere possibility.
- Unsupported reasoning must be labeled `inference`.
