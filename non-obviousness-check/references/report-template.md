# Inventive Step Report Template

Load this template only when the user wants a formal memo or saved report.
Use it to expand `inventive_step_report.md` into a formal memo.

## Required Sections

1. Matter under review
2. Target claim(s), jurisdiction, and legal frame
3. D1 and distinguishing features
4. `claim_diff_matrix.md` summary
5. Candidate combination paths
6. Motivation analysis
7. Secondary considerations
8. Conclusion
9. Facts / inferences / unknowns
10. Open risks and next actions

## Executive Summary

Include:

- matter under review
- target claim(s)
- jurisdiction
- selected framework
- strongest D1
- best combination path
- final result: `strong`, `weak`, or `uncertain`
- one-sentence reason

## Citation Format

Use one compact location format throughout:

- `US20240123456A1, col. 5, lines 12-20`
- `EP3456789A1, para [0042]`
- `CN114567890A, Fig. 3`
- `Comparative data table, filed declaration dated 2025-01-12`

## Combination Table

| Path | Difference supplied | Motivation | Compatibility | Expectation of success | Counterpoint | Strength |
| --- | --- | --- | --- | --- | --- | --- |
| D1 + D2 | feature F3 | explicit hint | compatible | supported | none | high |
| D1 + D3 | feature F4 | same field only | mixed | uncertain | weak compatibility | low |

## Conclusion Block

Always separate:

- facts
- inferences
- unknowns

Required wording:

- `Conclusion: strong. Under [jurisdiction], the best combination path is [path], supported by [motivation] and [expectation-of-success basis].`
- `Conclusion: weak. Under [jurisdiction], a combination path exists, but the case remains vulnerable because [weak link].`
- `Conclusion: uncertain. Under [jurisdiction], the current record does not support a definitive inventive-step conclusion because [blocker].`

State whether the inventive-step challenge is strong, weak, or uncertain under the selected jurisdiction.
Name the best path or the blocker explicitly.
