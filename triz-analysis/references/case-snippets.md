# Case Snippets (Balanced Mix)

Use these as starter patterns, not fixed answers.

## A. Enterprise Engineering Cases

### A1: Edge Device Thermal vs Performance
- Problem: inference latency is high; boosting frequency increases thermal throttling.
- Contradiction: performance up vs thermal reliability down.
- TRIZ direction: segmentation + dynamics + prior action.
- Option hint: dynamic workload partition + predictive thermal scheduling.
- Validation hint: A/B test latency-throttle curve under peak load.

### A2: Manufacturing Yield vs Throughput
- Problem: tighter quality checks reduce output.
- Contradiction: quality up vs throughput down.
- TRIZ direction: local quality + feedback + intermediary.
- Option hint: inline adaptive sampling and selective high-resolution inspection.
- Validation hint: pilot on one line with defect escape tracking.

### A3: Battery Life vs Device Responsiveness
- Problem: aggressive sleep strategy hurts UX responsiveness.
- Contradiction: energy efficiency up vs response time down.
- TRIZ direction: preliminary action + separation by condition.
- Option hint: context-aware wake policy by user state.
- Validation hint: field trial on battery drain and tap-to-response metric.

## B. Research Project Cases

### B1: Catalyst Activity vs Stability
- Problem: higher activity formulations degrade quickly.
- Contradiction: performance up vs durability down.
- TRIZ direction: composite materials + local quality.
- Option hint: graded structure catalyst with protected active sites.
- Validation hint: accelerated aging with activity-retention curves.

### B2: Model Accuracy vs Interpretability
- Problem: black-box model wins accuracy but is hard to explain.
- Contradiction: predictive power up vs explainability down.
- TRIZ direction: nested doll + intermediary + another dimension.
- Option hint: two-stage model (interpretable gate + high-capacity expert).
- Validation hint: compare AUC and explanation consistency on holdout sets.

### B3: Sensor Sensitivity vs Noise Robustness
- Problem: higher gain captures weak signals but increases false alarms.
- Contradiction: sensitivity up vs specificity down.
- TRIZ direction: feedback + parameter changes + partial/excessive action.
- Option hint: adaptive gain with uncertainty-triggered filtering.
- Validation hint: controlled signal/noise sweep and ROC shift analysis.

## C. Teaching/Learning Project Cases

### C1: Capstone Ambition vs Delivery Risk
- Problem: students pick ambitious designs and miss deadlines.
- Contradiction: innovation ambition up vs completion probability down.
- TRIZ direction: segmentation + prior action.
- Option hint: phased prototype milestones with risk gates.
- Validation hint: milestone completion ratio across cohorts.

### C2: Hands-on Depth vs Course Time
- Problem: deeper labs exceed available class hours.
- Contradiction: learning depth up vs schedule fit down.
- TRIZ direction: separation in time + self-service.
- Option hint: flipped-lab modules with pre-lab simulation.
- Validation hint: pre/post concept test plus lab completion time.

### C3: Team Collaboration vs Individual Assessment Fairness
- Problem: team projects hide individual contribution variance.
- Contradiction: collaboration benefits up vs individual fairness down.
- TRIZ direction: universality + feedback + local quality.
- Option hint: role-rotating tasks with contribution logging rubric.
- Validation hint: peer-review variance and rubric alignment audit.
